But the worker's terminal writes — markDelivered, markFailed, rescheduleDelivery — update by id with no status guard:

UPDATE deliveries SET status = 'delivered', ... WHERE id = ?

That backstop is what makes the test green.

Swap in a fast endpoint and the guarantee breaks: worker claims → gets 200 → markDelivered overwrites the cancel → the delivery ends delivered despite the endpoint being deleted. The PLAN edge case ("soft-delete → in-flight Deliveries → canceled, not failed") is only half-honored.

on markDelivered, markFailed, and rescheduleDelivery. Then a delivery that was canceled mid-flight stays canceled — the worker's write becomes a no-op.
