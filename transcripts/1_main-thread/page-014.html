🔴 Bug: successRate is wrong — always returns 1.0 (or null)

Two compounding mistakes in the query/logic:

const result = await db.execute({
  sql: `SELECT COUNT(*) AS n FROM deliveries
        WHERE status IN ('delivered','failed','canceled') AND updated_at >= ?`,  // only selects `n`
  args: [window],
});
const total = Number(result.rows[0]?.n ?? 0);
const failed = Number(result.rows[0]?.failed ?? 0);  // ← no `failed` column in the query → always 0
if (total + failed === 0) return null;
return total / (total + failed);                      // ← total / (total + 0) === 1.0
  1. failed is read from a column that doesn't exist (AS n is the only projection), so it's always 0.
  2. canceled is counted in total, but the design says success rate is delivered ÷ (delivered + failed) — canceled must be excluded (nothing failed).

Net: whenever ≥1 terminal delivery exists, it returns total/total = 1.0; otherwise null. The metric is meaningless. My runtime check with 1 delivered / 1 failed / 1 canceled returned 1 (should be 0.5).

Fix — count the two states separately and exclude canceled:

export async function successRate(windowMs = DEFAULT_WINDOW_MS): Promise<number | null> {
  const window = new Date(Date.now() - windowMs).toISOString();
  const result = await db.execute({
    sql: `SELECT
        SUM(CASE WHEN status = 'delivered' THEN 1 ELSE 0 END) AS delivered,
        SUM(CASE WHEN status = 'failed'    THEN 1 ELSE 0 END) AS failed
      FROM deliveries
      WHERE status IN ('delivered','failed') AND updated_at >= ?`,
    args: [window],
  });
  const delivered = Number(result.rows[0]?.delivered ?? 0);
  const failed = Number(result.rows[0]?.failed ?? 0);
  if (delivered + failed === 0) return null;
  return delivered / (delivered + failed);
}
